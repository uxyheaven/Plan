# Plan

这个项目现在定位为一个以 Markdown 为主的内容工作区，主要服务两类任务：

1. 方案设计
2. 演讲稿讨论与打磨

我可以基于这里的文档继续和你协作，包括梳理结构、补全内容、改写表达、拆分章节、生成提纲和逐字稿。

## 当前目录

### `designs/`

存放方案设计相关内容，包括：

- 某个专题或项目的完整设计资料
- 概览版方案
- 设计模板

### `speeches/`

存放演讲稿相关内容，包括：

- 演讲主题草稿
- 结构设计
- 开场稿、逐字稿
- 演讲模板

## 推荐使用方式

### 做方案设计时

- 先在 `designs/templates/design-template.md` 复制一个新文档
- 如果是长期项目，单独建一个目录，按“概览 / 规范 / Prompt / 关键要点”拆分
- 如果只是一次性思路整理，先写一个单文件版本，再决定是否拆目录

### 做演讲稿时

- 先在 `speeches/templates/speech-template.md` 复制一个新文档
- 先确定听众、时长、目标，再写结构
- 逐步从提纲推进到开场稿、关键段落、完整逐字稿

## 当前内容索引

### 方案设计

- `designs/ai-analysis-system/`：AI 分析系统专题设计文档

### 演讲稿

- `speeches/ordinary-people-investing.md`：普通人如何投资

## 协作约定

- 优先使用 Markdown
- 一个文档只解决一个明确问题
- 根 README 只做导航，不承载大段正文
- 如果某个主题持续演化，就升级为独立目录

## 后续新增建议

- 新方案放进 `designs/`
- 新演讲稿放进 `speeches/`
- 尽量使用统一文件结构，方便我后续直接续写和重构

# Plan 仓库说明

这个仓库用于存放**多个系统的方案设计文档**。

## 目录约定
- `plans/`：各系统的独立方案文档（按主题拆分，便于持续新增）。

## 当前方案列表
- AI 分析系统设计方案：`plans/ai-analysis-system.md`

## 维护建议
- 新增一个系统方案时，创建新的 Markdown 文件并放入 `plans/`。
- 保持 README 作为总览与索引，不承载完整方案正文。

## 冲突处理约定
- 若多人同时修改 `README.md`，以“保留索引能力、避免放入长正文”为合并准则。
- 发生冲突时优先保留双方新增的方案链接，并按文件名排序。
- 方案正文冲突优先在对应 `plans/*.md` 文件中解决，README 仅保留最终链接。

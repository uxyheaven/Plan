# 分析洞察系统（Analysis Insights）

基于AI能力的自主分析系统，支持多场景数据分析、智能规划和用户交互。

## 🎯 核心特点

✅ **多场景支持** - 通过配置化的Scenario实现快速接入  
✅ **自主规划** - LLM生成分析计划，智能编排执行步骤  
✅ **高度可扩展** - Skill库 + 框架库 + 数据源解耦  
✅ **保证准确性** - LLM仅负责推理，所有计算由系统完成  
✅ **完整溯源** - 每条数据都能追踪来源和处理链路  
✅ **优雅降级** - 多层容错机制确保用户体验

## 🔑 关键设计决策

### 1️⃣ LLM在系统中的角色

**问题**：LLM在数值计算上容易出错，如何确保准确性？  
**解决方案**：LLM专注于**推理、规划、解释**，所有数值计算由系统完成  
**实现**：

- 调用1: 分析计划生成（LLM规划）
- 调用2-3: 洞察提取和报告生成（LLM推理）
- 调用4: 用户问答（LLM综合）

### 2️⃣ 多数据源的一致性和可信度

**问题**：数据来自多个系统，如何确保可信度？  
**解决方案**：不记录数据，只记录来源和处理链路  
**实现**：

```json
{
  "metric": "转化率",
  "value": 2.5,
  "metadata": {
    "source": "order_system",
    "timestamp": "2024-03-17 10:30",
    "confidence": 0.99,
    "lineage": [...]  // 完整处理链路
  }
}
```

### 3️⃣ 从LLM生成的分析计划可能有问题

**问题**：LLM生成的分析计划可能选错Skill或框架  
**解决方案**：多层验证 + 降级方案  
**实现**：

```
Schema验证 → 逻辑验证 → 执行验证 → 降级到预设计划
```

## 💡 核心概念

### Scenario（场景）

特定业务问题的分析配置包，包含：

- 分析思路
- 指定的Skills列表（并行执行）
- 指定的分析框架（本地处理）
- 数据源配置

示例：`scenarios/conversion_analysis.yaml`

### Skill（技能）

原子级的数据获取/分析能力，特点：

- **独立性**：独立开发、测试、部署
- **复用性**：多个场景共用一个Skill
- **并行性**：多个Skill可并行执行
- **溯源性**：返回标准格式+来源元数据

### Framework（分析框架）

结构化的思维方式，支持：

- **5W2H** - 多维度问题诊断
- **SWOT** - 竞争力分析
- **对标分析** - 与竞品对比

**关键特点**：框架处理是本地计算，不涉及LLM调用

## 🛡️ 容错与降级

系统支持分层降级策略：

| 降级级别 | 场景           | 处理方式                     |
| -------- | -------------- | ---------------------------- |
| 级别1    | Skill超时      | 使用缓存数据 + 继续其他分析  |
| 级别2    | 部分框架不适用 | 使用可用框架 + 缩小结果范围  |
| 级别3    | 核心Skill失败  | 中断分析 + 展示错误信息      |
| 级别4    | LLM调用失败    | 仅展示数据+图表 + 禁用AI解读 |

## ✅ 最核心的3个问题

### ❓ Q1: 如何确保LLM生成的分析计划正确？

→ **多层验证机制**（见 spec.md 第七部分）

- Schema验证 + 逻辑验证 + 执行验证 + 人工降级

### ❓ Q2: 如何追踪多源数据的来源？

→ **元数据标注 + 完整链路记录**（见 spec.md 第七部分）

- 每条数据 + source + timestamp + confidence + lineage

### ❓ Q3: 如何优雅处理分析失败？

→ **分层降级策略**（见 spec.md 第七部分）

- 缓存降级 → 部分分析 → 中断降级 → 图形降级
